# Editor Product Model

The editor product is designed to feel externally comparable to a high-end AAA editor:
- main shell
- multiple dockable views
- one or more 3D viewports
- viewport toolbar and overlays
- outliner and world browser
- content browser
- details/inspector
- mode-specific tool shelves
- diagnostics and output surfaces
- assistant dock
- play/simulate/debug controls
- build and release center

## Product loops
- open project -> open workspace -> inspect world -> edit entities -> preview -> simulate -> validate -> build -> release
- import content -> organize assets -> assign materials -> place into world -> run diagnostics -> publish artifact
- ask assistant -> inspect proposal -> lower to legal commands -> apply/revert -> validate and continue

## Completeness law
The editor product is not closed until a full project can be created, edited, simulated, debugged, packaged, and reviewed without leaving the editor product surface.
