# Content and Asset Model

## Browser requirements
- folders and virtual collections
- tags, filters, search, saved searches
- dependencies and reverse dependencies
- preview thumbnails and live preview panes
- import queues
- metadata and version surfaces
- drag/drop into viewport, outliner, or suites
- validation warnings on assets
- source/import origin visibility

## Asset classes
- meshes
- materials and textures
- sounds and voices
- animations
- particles and VFX assets
- scripts and graphs
- UI assets
- world templates
- blueprints/prefabs/archetypes equivalent
- suite-specific authored packages

## Law
The browser is a view and routing surface.
Canonical asset truth still lives below the product layer.
