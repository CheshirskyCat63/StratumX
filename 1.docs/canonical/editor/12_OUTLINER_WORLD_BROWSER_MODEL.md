# Outliner and World Browser Model

## Outliner requirements
- hierarchical scene/object tree
- filters and search
- visibility and lock toggles
- grouping and folders
- selection sync with viewport and details
- context actions
- diagnostics badges

## World browser requirements
- partition cells / regions / zones
- streaming state views
- world composition and sub-world management
- residency and load hints
- large-world navigation map
- authoring ownership markers
- simulation/state overlays

## Law
Outliner focuses scene hierarchy.
World browser focuses world topology and partitioning.
They may cross-highlight each other but remain distinct surfaces.
