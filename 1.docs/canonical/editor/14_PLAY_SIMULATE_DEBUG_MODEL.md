# Play, Simulate, and Debug Model

## Runtime modes
- edit
- simulate in editor
- play in editor
- paused play
- detached observer/debug camera
- replay/debug stepping where available

## Debug surfaces
- output log
- budget and performance charts
- simulation event trace
- AI/debug overlays
- partition/residency overlays
- collision/navigation overlays
- replication/network overlays if present
- render pass and culling overlays
- memory/GPU pressure views

## Law
Play and simulate are product modes above lower-stack runtime control.
They may not bypass lower-stack legality or invent their own state authority.
