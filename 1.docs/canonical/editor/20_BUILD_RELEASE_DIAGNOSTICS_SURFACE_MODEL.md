# Build, Release, and Diagnostics Surface Model

## Diagnostics responsibilities
- warnings, errors, metrics, traces, and budget projections
- validation findings and health views
- hotspot and breakdown surfaces

## Build/release responsibilities
- build graph and task visibility
- package preset invocation
- release checklist visibility
- artifact, target, and package status projection

## Laws
- diagnostics views are projections over lower-stack evidence
- build and release surfaces do not own packaging logic
