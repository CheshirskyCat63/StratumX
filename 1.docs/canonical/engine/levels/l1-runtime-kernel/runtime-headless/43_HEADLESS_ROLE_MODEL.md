# Headless Role Model

## Role

Role contract for `headless-20`.

## Data Model

Headless owns authoritative progression without local presentation ownership.
It may host remote sessions, replay services, and storage verification only within the headless mixed-pressure proof.

## Role Matrix

| Role | Legal ownership | Reserved ceiling source | Illegal ownership |
|---|---|---|---|
| authoritative host | world progression, publication windows, replay cadence | headless profile composition proof | presentation path |
| dedicated server | same as authoritative host with no local presentation reserve | headless profile composition proof | local-frame semantics |
| offline validation headless role | bounded startup/replay/storage validation | startup + replay law | widening into general diagnostics mode |

## Failure Posture

- headless roles may not inherit local-presentation reserve;
- host burden may not exceed the mixed-pressure headless proof;
- verification and diagnostics yield before authoritative progression.

## Illegal Patterns

- presentable frame ownership;
- borrowing realtime queue semantics;
- host burden that exceeds headless proof;
- hidden role widening after bootstrap.

## Role-Bind Matrix

| Bind concern | Requirement | Illegal posture |
|---|---|---|
| startup role bind | explicit and frozen before launch | implicit host widening |
| publication entitlement | bounded by selected headless role | borrowing presentation reserve |
| verification posture | bounded auxiliary ownership only | validation role becoming general runtime mode |

## Verification Boundaries

Headless validation roles may inspect, checkpoint, or verify only inside bounded auxiliary windows.
They may not become a second owner of cadence, network transport, or storage governance.

## Local Operating Law

`headless-20` owns authoritative progression without local presentation semantics.
Every local headless contract must preserve that asymmetry.
