# Profile Model

## Role

Runtime profile constitution.

## Canonical Definition

`Profile Model` is a canonical element of `engine_runtime` inside `L1. Runtime Kernel`. It exists as one explicit part of the engine stack and does not transfer its ownership class into another crate or level.

## Data Model

Numeric Authority: STRATUMX_NUMERIC_SOURCE_OF_TRUTH.md §3, §5, §6, §7, §9, §10, §12


Shared profile model binding core runtime law to headless and realtime variants.
Profiles are not cosmetic names. They freeze cadence, queue posture, memory/residency envelope, and legal service burden.
Numeric posture is frozen by `STRATUMX_NUMERIC_SOURCE_OF_TRUTH.md` and `STRATUMX_ABSOLUTE_BUDGET_CONSTITUTION.md`.

## Canonical Profile Envelope

| Profile | Queue posture | Memory / residency envelope |
|---|---|---|
| `interactive-60` | low-latency presentation queue <= 1 from numeric source | uses `interactive-60` RAM and VRAM ceilings from `STRATUMX_ABSOLUTE_BUDGET_CONSTITUTION.md` |
| `listen-host-60` | low-latency presentation queue <= 1 plus bounded host publication burden from numeric source | uses `listen-host-60` RAM and VRAM ceilings from `STRATUMX_ABSOLUTE_BUDGET_CONSTITUTION.md` |
| `headless-20` | no presentation queue | uses `headless-20` RAM ceilings from `STRATUMX_ABSOLUTE_BUDGET_CONSTITUTION.md` and `STRATUMX_MEMORY_AND_RESIDENCY_CONSTITUTION.md` |
