# Boundary Preservation

## Canonical Position

`engine_content` is preserved as a standalone crate and moved above the earlier L2 grouping.

## Earlier Boundary Relation

Earlier canon grouped content with presentation/resource domains inside L2. Master canon keeps the crate but reclassifies it as `L3.2. Resource Systems`.

## Reason

Content processing is resource-facing pipeline work, not critical world progression.

## Runtime-Pack Relation

Runtime resource services may consume runtime-pack inputs shaped by `engine_content`, but that relation is startup-mediated or externally prepared and does not create a direct `L1.5 -> L3.2` crate dependency.
