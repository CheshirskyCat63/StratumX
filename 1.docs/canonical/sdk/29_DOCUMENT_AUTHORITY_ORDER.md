# Document Authority Order

## Authority order
1. root documents
2. constitutions
3. `00_LEVEL.md`
4. local layer docs

## Conflict resolution
- a lower document may narrow authority
- a lower document may clarify terminology
- a lower document may never widen owned data kind, role class, or allowed dependency scope against a higher document

## Sealing law
A package is sealed only when the authority chain is contradiction-free.
