# Degradation Policy Model

Pressure-first degradation law:
- far, decorative, optional, report-only, and expensive convenience work degrades first
- preview and assistant speculation degrade before authority correctness
- indices and derived projections may defer rebuild before truth is corrupted
- near, gameplay-critical, transaction-correct, legality-critical, and manifest-integrity work degrades last
- legality, transaction correctness, and manifest integrity never degrade

Degradation must be explicit by domain, by profile, and by plane. Hidden fallback code is forbidden.
