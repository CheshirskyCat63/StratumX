# Communication Model

## Canonical Rule

Communication is built on:

- types;
- descriptors;
- views;
- handles;
- snapshots;
- staged outputs;
- synthesis products;
- diagnostics.

## Stack Flow

- lower substrate provides structure;
- world provides authoritative truth;
- material provides shared property truth;
- runtime provides execution windows;
- critical simulation families compute over world truth;
- model systems consume structured context and emit structured outputs;
- synthesis systems emit image and acoustic products;
- resource systems emit prepared resource products;
- startup assembles and launches the stack.

## Ownership Rule

- world truth ownership stays in `engine_world`;
- global parallel ownership stays in `engine_runtime`;
- startup owns launch, not execution;
- upper families emit staged or synthesized products, not arbitrary world truth.
