# Boundary Preservation

## Widened Replacement

`engine_generation` replaces the earlier standalone `engine_dialogue` boundary.

## Canonical Meaning

Dialogue generation is preserved as one generation use-case, but the crate boundary is widened to cover structured generation beyond dialogue alone.

## Reason

The master canon needs one scalable generation boundary that can serve runtime generation, content generation, and future tool-side generation without fragmenting into narrow special-purpose crates.
